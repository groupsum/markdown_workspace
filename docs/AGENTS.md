# Repository Docs Agent Instructions

Repository docs must distinguish architecture, conformance, current-state reports, operations, package references, and user-facing product guidance.

- Use `content_seo_specialist` for product-facing docs and `ssot_governance` for ADR, SPEC, feature, claim, evidence, boundary, release, or certification docs.
- Keep docs grounded in repo truth and cite commands, package names, file paths, and generated evidence where relevant.
- Do not let compliance projections replace `.ssot` authority; use `ssot-registry` for SSOT changes.
- Keep current-state docs clearly dated when they describe a snapshot.
