# Markdown Support

MdWork focuses on practical Markdown authoring for technical notes, docs, and project writing.

## Core Syntax
Use standard Markdown for headings, paragraphs, links, emphasis, blockquotes, code, and lists.

```markdown
# Release Notes

MdWork keeps drafts local by default.
```

## GitHub-Flavored Markdown
MdWork supports common GitHub-Flavored Markdown patterns, including tables and task lists.

```markdown
| Area | Status |
| :--- | :--- |
| Local storage | Ready |
| GitHub sync | Optional |

- [x] Draft locally
- [ ] Sync when ready
```

## Code Blocks
Fence code blocks with a language name to preserve readable source snippets.

````markdown
```ts
const storageMode = "local-first";
```
````

## Production Guidance
Keep documents portable. Prefer Markdown syntax that renders cleanly across MdWork, GitHub, npm, and static documentation sites.
