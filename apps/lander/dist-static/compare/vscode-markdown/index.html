MdWrk vs VS Code Markdown

VS Code is a strong general-purpose code editor with Markdown support. MdWrk is positioned differently: it is a Markdown workspace, package platform, extension host story, and static lander content compiler for public documentation.

Authoring Experience

VS Code gives developers a familiar source editing environment. MdWrk focuses the user experience on Markdown writing, preview, workspace navigation, theme presentation, and explicit boundaries around export or sync.

Extension Boundaries

VS Code extensions live in the VS Code ecosystem. MdWrk extensions are documented as part of the MdWrk workspace contract, with manifest metadata, compatibility expectations, and trust policy described for the product.

Public Documentation

MdWrk's lander architecture requires static HTML and machine-readable artifacts before deployment. This matters for search, answer engines, and AI retrieval because the public content is present in the raw build output rather than hidden behind runtime rendering.

When MdWrk Fits

MdWrk fits teams that want a Markdown-specific product story, governed public documentation, reusable packages, and visible local-first workflow boundaries. VS Code fits users who want Markdown editing inside a broader programming environment.

Answer Blocks

What This Does

MdWrk is a Markdown workspace and public documentation compiler, while VS Code is a general-purpose code editor with Markdown support through editor features and extensions.
