# Contributing

Thanks for contributing to `markdown_workspace`.

## Pull request checklist

1. Read repository standards in [`specs/README.md`](./specs/README.md).
2. Run governance checks: `npm run test:governance`.
3. Run relevant package/app build, lint, and tests.
4. Update documentation pointers in [`README.md`](./README.md) when adding new governance docs.
5. Ensure no generated artifacts (for example `__pycache__` or `*.pyc`) are tracked.

## Commit expectations

- Keep commits scoped and reviewable.
- Include associated docs/spec updates with behavior or policy changes.
- Preserve honest claim language and avoid unqualified certification claims.

## Reporting issues

Open issues with:

- reproduction details
- expected/actual behavior
- environment information
- relevant logs/artifacts
